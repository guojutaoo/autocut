# AutoCut 架构升级需求：多模态情境剧本 (Multi-modal Contextual Transcript)

## 1. 核心目标 (Core Goal)
通过提取视频的视觉、听觉和语速特征，为 LLM 提供精准的“场景还原”坐标，使其在“看不见视频”的情况下，能写出极具画面张力和节奏感的解说词。

## 2. 文案结构规范 (Output Format)
每一行（或每一个片段）的输出必须遵循以下精简、高信息密度的结构：

> **`[开始时间 - 结束时间] 台词：“{台词内容}” | 特征：【视觉：{视觉标签}】 | 【听觉：{听觉标签}】 | 【节奏：{语速标签}】`**

### 示例展示：
- `[00:15:30,000 - 00:15:45,500] 台词：“儿臣知罪。” | 特征：【视觉：静态/阴暗】 | 【听觉：死寂/压抑】 | 【节奏：语速迟缓】`
- `[00:22:10,000 - 00:22:18,000] 台词：“杀！” | 特征：【视觉：剧烈晃动/高能】 | 【听觉：能量爆发/嘈杂】 | 【节奏：语速极快】`

---

## 3. 特征定义与提取逻辑

### A. 视觉特征 (Visual Vibe)
- **静态/平稳**：画面变动极小（如对话戏、空镜头）。
- **剧烈晃动/高能**：画面标准差（Std Dev）瞬时飙升（如打斗、奔跑、镜头切断）。
- **阴暗/明亮**：根据平均像素亮度（Brightness）区分深夜、室内或白昼场景。

### B. 听觉特征 (Audio Vibe)
- **死寂/压抑**：RMS 能量持续低于极低阈值（如悬疑、对峙、沉思时刻）。
- **能量爆发/嘈杂**：RMS 能量瞬时超过高阈值（如怒吼、爆炸、背景音量骤增）。
- **正常/对话**：音频能量处于常规波动区间。

### C. 语速特征 (Speech Pace)
- **语速迟缓/沉重**：每秒字数（CPS）低于阈值。通常意味着深思熟虑、威胁或极度悲伤。
- **语速极快/紧张**：CPS 显著高于平均值。通常意味着争执、焦虑或紧急情况。
- **正常/平稳**：常规语速。

---

## 4. 物理剪辑策略与 LLM 指令规范 (Render Strategies)

为了提升成片的导演感和视觉节奏，系统支持以下四种**物理剪辑策略**。LLM 必须在 JSON 中通过 `render_type` 字段明确指定：

| 策略标识 (`render_type`) | 物理表现 (Rendering) | 适用场景建议 |
| :--- | :--- | :--- |
| **`freeze`** (默认) | **前置视频 -> 定格画面 + 旁白 -> 后置视频** | 核心权谋剖析、心理战、细节拆解。 |
| **`overdub`** | **视频正常播放 + 旁白覆盖 (Ducking 混音)** | 交代背景、转场、非核心对话的快速串联。 |
| **`pure_audio`** | **原速播放 + 100% 原声 (无旁白)** | 角色金句爆发、震撼的大场面原声呈现。 |
| **`slow_mo`** | **0.5倍慢动作播放 + 旁白覆盖** | 微表情捕捉、极度压抑或悬疑的氛围渲染。 |

---

## 5. 针对 LLM Skill 的更新规范

### A. 格式要求 (JSON Schema)
LLM 输出的每一段 `segment` 必须包含 `render_type`：
```json
{
  "index": 0,
  "start": 12.5,
  "end": 25.0,
  "render_type": "overdub", 
  "narration_text": "此时的四阿哥已经察觉到了局势的变化..."
}
```

### B. 策略决策逻辑
- **决策依据**：LLM 应结合 `environment_metadata` 进行判断。
    - 若 `visual_vibe` 为“静态”且有深度台词 -> 选 `freeze`。
    - 若 `visual_vibe` 为“剧烈变化”且需要解说 -> 选 `overdub` 或 `slow_mo`。
    - 若 `audio_vibe` 为“能量爆发”且台词极佳 -> 选 `pure_audio` 留白。

## 6. 深度生产规范与质量守底 (Production Guardrails)

为了确保 LLM 生成的剧本能直接进入“工业化生产”并产出高质量视频，LLM 必须遵循以下深度规范：

### A. 锚点与画面对位 (Anchor Precision)
- **强制锚点回传**：每一个片段必须指定一个 `anchor_time`（定格/解说的核心时刻）。LLM 选取的 `narration_text` 描述内容必须与该时刻的画面严格对位。
- **动作单一化**：严禁在一段解说词中描述一连串动作（如“四阿哥进殿并下跪”）。如果需要描述多个动作，请拆分为多个连续的片段。

### B. 音画衔接与氛围 (Audio & Atmosphere)
- **BGM 连续性**：系统将采用全局 Cross-fade 或统一垫乐逻辑。LLM 在编写解说词时，应预留 1-2 秒的“呼吸位”，避免音频断层感。
- **策略与画面联动**：
    - `freeze`：仅限对单一表情、细节、静态局势的深度拆解。
    - `overdub`：用于描述动态过程、环境交代、逻辑串联。

### C. 视觉审美与字幕安全 (Visual Safety)
- **字幕防冲突**：旁白字幕将统一采用“深色背景半透明条 + 白字”样式，且位置上移至画面中下部，以避开原片自带的底部字幕。
- **主体意识**：LLM 选取的 `anchor_time` 对应的画面主体应尽量位于中心区域，以适配 9:16 的竖屏裁剪。
