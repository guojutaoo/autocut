# 叙事承接与锚点铺垫修复方案

## 目标
解决 LLM 输出剧本缺少承上启下与锚点内容铺垫的问题，让旁白在新人物出场和 freeze/slow_mo 等锚点段落前有明确引入与交代。

## 实施步骤
1. **梳理现有生成链路与输入上下文**  
   - 读取当前用于 LLM 的输入（`transcript_for_llm.txt`、`compose_plan.json`）与输出（`video_script_expert_output.json`），确认哪些字段缺失“桥接信息”和“锚点前台词交代”。

2. **更新 skill 规则（video-script-expert.md）**  
   - 新增“承上启下桥接句”的强制规则：当片段涉及新人物/新场景时，必须用一句话完成时空切换与人物引入。  
   - 新增“锚点铺垫规则”：`start~anchor_time` 之间必须交代锚点处的关键台词或动作含义，不允许直接进入分析。  
   - 明确输出需包含“桥接句 + 锚点交代句 + 解析句”的结构模板。

3. **在生成产物中落地结构化字段（可选增强）**  
   - 如果需要更强约束，在输出 JSON 中新增：  
     - `bridge_text`：承上启下句  
     - `anchor_context`：锚点前交代句  
   - 并在渲染侧将其拼接到 `narration_text`，保证输出必包含这两类句子。

4. **重跑剧本生成并人工抽检**  
   - 使用更新后的 skill 重新生成剧本。  
   - 抽检包含“人物切换”和“freeze/slow_mo”片段，确认出现桥接与锚点交代。

## 验收标准
- 任何新人物/新场景进入时，旁白都有明确的“时空切换 + 人物引入”句。  
- freeze/slow_mo 片段中，旁白在分析前先交代锚点台词/动作含义。  
- 输出不再出现“直接开始说四爷说了什么”或“未交代锚点内容”的割裂感。
